# LIMS Configuration Management - Ansible

This directory contains the Ansible playbooks and roles to configure the provisioned EC2 instances for the LIMS application.

## Prerequisites

- Ansible installed
- EC2 instance provisioned by Terraform (Terraform automatically generates the inventory file here)
- SSH private key (`AWS-Key.pem`) available on your local machine

## Roles

- **docker:** Installs Docker Engine, Docker Compose plugin, and configures the `ubuntu` user.
- **nginx:** Installs the Nginx web server.
- **certbot:** Installs Certbot and the Nginx plugin for SSL certificate generation.

## Execution Order

1. **Verify Inventory:**
   Ensure the inventory file (e.g., `inventory/dev.ini`) was generated by Terraform and contains the correct IP.

2. **Update SSH Key Path:**
   Ensure your SSH key is available at `~/.ssh/AWS-Key.pem` or update the `private_key_file` path in `ansible.cfg`.

3. **Run the Playbook:**
   Run the setup playbook against the desired environment inventory:
   ```bash
   ansible-playbook -i inventory/dev.ini playbooks/setup.yml
   ```

   **Run Specific Tags (Optional):**
   If you only want to run specific parts of the configuration:
   ```bash
   ansible-playbook -i inventory/dev.ini playbooks/setup.yml --tags docker
   ansible-playbook -i inventory/dev.ini playbooks/setup.yml --tags nginx
   ```
